发布到市场时，必须保证 `https://zonya.work/releases/` 下同时存在安装包和 `latest.yml`，否则桌面端自动更新不会工作。
1. 首次安装 用户从下载页获取 Windows 安装包，完成首次安装并进入登录页。
2. 自动检测 桌面端启动后自动检查 `https://zonya.work/releases/latest.yml` 是否有更高版本。
3. 后台下载 发现新版本后提示用户，确认后后台下载更新包，不打断当前业务操作。
4. 下次启动安装 下载完成后，用户退出并重新打开应用时自动安装更新，这是 PC 端最稳妥的体验。

发布前必须完成

  • 桌面应用必须用 `electron-builder` 产出 Windows NSIS 安装包。
  • 必须配置有效 HTTPS，否则安装包下载和自动更新都会失去信任。
  • 必须把安装包和 `latest.yml` 上传到 `https://zonya.work/releases/`。
  • 最好补代码签名，否则 Windows SmartScreen 会弹更多风险提示。

适合你的更新策略

  • 启动后自动检查更新。
  • 发现更新后主动提示用户。
  • 下载完成后允许“立即安装”或“下次启动安装”。
  • 不要把更新绑定在账号登录上，更新是应用级，不是用户级。

市场发布注意事项

  • 下载页文案要明确“Windows 桌面版”，避免用户误以为是网页应用。
  • 版本号、发布日期、更新日志要跟随每次发布同步更新。
  • 如果面向企业客户，建议保留离线安装包和版本回滚包。
  • 发布前至少做一次完整升级测试：`1.0.0 -> 1.0.1`。

当前结论

现在的仓库已经补上了桌面端自动更新骨架，但还没有真正生成 Windows 安装包，也还没有把 `releases/latest.yml` 和安装包部署到线上。 如果你要正式向市场发布，下一步应该先完成本地打包、生成发布物、上传到 `https://zonya.work/releases/`，再做一次真实升级测试。
