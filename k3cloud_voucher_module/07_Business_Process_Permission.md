# 金蝶云星空 - 会计凭证管理模块 - 业务流程与权限控制

## 一、凭证业务流程设计

### 1.1 标准业务流程图

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  制单   │ →  │  审核   │ →  │  记账   │ →  │  结账   │
│ (保存)  │    │         │    │         │    │         │
└────┬────┘    └────┬────┘    └────┬────┘    └─────────┘
     │               │               │
     ├─ 编辑修改      ├─ 反审核       └─ 反记账
     ├─ 删除          │
     └─ 提交审核      └─ 退回制单
```

### 1.2 业务流程详细说明

#### 阶段一：凭证制单
| 操作项 | 说明 | 前置条件 | 后续状态 |
|-------|------|---------|---------|
| 新增凭证 | 创建新凭证 | 无 | 暂存 |
| 保存凭证 | 保存暂存凭证 | 暂存 | 制单 |
| 编辑凭证 | 修改凭证内容 | 制单 | 制单 |
| 删除凭证 | 删除凭证 | 制单 | 已删除 |
| 复制凭证 | 复制生成新凭证 | 任意状态 | 制单 |

#### 阶段二：凭证审核
| 操作项 | 说明 | 前置条件 | 后续状态 |
|-------|------|---------|---------|
| 审核凭证 | 审核通过 | 制单 | 已审核 |
| 反审核 | 取消审核 | 已审核 | 制单 |
| 批量审核 | 批量审核通过 | 制单 | 已审核 |

#### 阶段三：凭证记账
| 操作项 | 说明 | 前置条件 | 后续状态 |
|-------|------|---------|---------|
| 记账 | 登记账簿 | 已审核 | 已记账 |
| 反记账 | 取消记账 | 已记账 | 已审核 |
| 批量记账 | 批量登记账簿 | 已审核 | 已记账 |

#### 特殊业务：凭证红冲
| 操作项 | 说明 | 前置条件 | 后续状态 |
|-------|------|---------|---------|
| 红冲 | 生成红字冲销凭证 | 已记账 | - |
| 部分红冲 | 部分金额红冲 | 已记账 | - |

## 二、状态机设计

### 2.1 单据状态定义

| 状态标识 | 状态名称 | 状态描述 | 允许操作 |
|---------|---------|---------|---------|
| Z | 暂存 | 凭证草稿 | 编辑、保存、删除 |
| A | 制单 | 已保存未审核 | 编辑、删除、提交审核、审核 |
| B | 已审核 | 审核通过 | 反审核、记账 |
| C | 已记账 | 已登记账簿 | 反记账、红冲、查询、打印 |
| D | 已作废 | 凭证已作废 | 查询 |

### 2.2 状态流转规则

```
┌──────┐
│ 暂存 │
└───┬──┘
    │ 保存
    ↓
┌──────┐
│ 制单 │←──────────┐
└───┬──┘           │ 反审核
    │ 审核         │
    ↓              │
┌────────┐         │
│ 已审核 │─────────┘
└───┬────┘
    │ 记账
    ↓
┌────────┐
│ 已记账 │←──────────┐
└────────┘           │ 反记账
    ↑                │
    │ 结账保护       │
    └────────────────┘
```

### 2.3 状态流转验证规则

#### 制单 → 已审核
- 校验借贷平衡
- 校验科目有效性
- 校验辅助核算完整性
- 校验现金流量分配（如需要）
- 校验期间未结账

#### 已审核 → 已记账
- 校验凭证已审核
- 校验期间未结账
- 校验科目未禁用
- 校验辅助核算余额

#### 已审核 → 制单（反审核）
- 校验凭证未记账
- 校验期间未结账
- 校验操作人权限

#### 已记账 → 已审核（反记账）
- 校验期间未结账
- 校验凭证未参与汇兑损益
- 校验凭证未参与期间损益结转
- 校验操作人权限

## 五、业务规则配置

### 5.1 校验规则

#### 规则一：借贷平衡校验
- **规则ID**：VOUCHER_BALANCE_CHECK
- **触发时机**：保存前、审核前
- **规则描述**：凭证借方金额合计必须等于贷方金额合计
- **错误提示**：借贷不平衡，请检查金额！

#### 规则二：科目有效性校验
- **规则ID**：VOUCHER_ACCOUNT_CHECK
- **触发时机**：保存前、审核前
- **规则描述**：科目必须存在且未禁用，且为最明细科目
- **错误提示**：科目【{0}】无效或不是最明细科目！

#### 规则三：辅助核算完整性校验
- **规则ID**：VOUCHER_AUX_CHECK
- **触发时机**：保存前、审核前
- **规则描述**：有辅助核算的科目必须录入辅助核算信息
- **错误提示**：科目【{0}】缺少辅助核算【{1}】信息！

#### 规则四：现金流量分配校验
- **规则ID**：VOUCHER_CASHFLOW_CHECK
- **触发时机**：保存前、审核前
- **规则描述**：现金类科目必须分配现金流量项目
- **错误提示**：现金科目【{0}】未分配现金流量项目！

#### 规则五：期间结账校验
- **规则ID**：VOUCHER_PERIOD_CHECK
- **触发时机**：所有操作
- **规则描述**：已结账期间不允许操作凭证
- **错误提示**：期间【{0}】已结账，不允许操作！

#### 规则六：制单与审核分离
- **规则ID**：VOUCHER_AUDIT_SEPARATE
- **触发时机**：审核前
- **规则描述**：制单人与审核人不能为同一人（可配置）
- **错误提示**：制单人与审核人不能为同一人！

### 5.2 自动规则

#### 规则一：自动编码
- **规则ID**：VOUCHER_AUTO_NUMBER
- **触发时机**：新增时
- **规则描述**：按凭证类型+期间+流水号自动生成凭证号
- **编码格式**：{类型}{年}{月}{流水号}

#### 规则二：默认凭证日期
- **规则ID**：VOUCHER_DEFAULT_DATE
- **触发时机**：新增时
- **规则描述**：默认凭证日期为系统当前日期

#### 规则三：默认会计期间
- **规则ID**：VOUCHER_DEFAULT_PERIOD
- **触发时机**：新增时
- **规则描述**：根据凭证日期自动确定会计期间

#### 规则四：自动平衡
- **规则ID**：VOUCHER_AUTO_BALANCE
- **触发时机**：金额改变时
- **规则描述**：根据已录入金额自动计算最后一行差额

#### 规则五：摘要自动携带
- **规则ID**：VOUCHER_SUMMARY_COPY
- **触发时机**：新增行时
- **规则描述**：新行摘要自动携带上一行摘要

## 六、审批流程配置

### 6.1 简单审批流程

#### 流程一：单人审核
```
制单 → 审核 → 记账
```
- 适用场景：小型企业，财务人员少
- 配置要点：审核人可为财务主管

### 6.2 复杂审批流程

#### 流程二：多级审核
```
制单 → 初审 → 复核 → 审核 → 记账
```
- 适用场景：大中型企业，财务管理规范
- 配置要点：
  - 初审：制单会计互审
  - 复核：主管会计审核
  - 审核：财务经理审批

#### 流程三：金额分级审批
```
制单 → 金额判断 → 分支审批
       ├─ ≤1万 → 主管会计审核
       ├─ ≤10万 → 财务经理审核
       └─ >10万 → 财务总监审批
```

### 6.3 审批条件配置

| 条件类型 | 条件说明 | 示例 |
|---------|---------|------|
| 金额条件 | 根据金额大小走不同分支 | 借方金额>10000 |
| 科目条件 | 根据科目类型走不同分支 | 科目包含"管理费用" |
| 部门条件 | 根据部门走不同分支 | 部门="销售部" |
| 制单人条件 | 根据制单人走不同分支 | 制单人="张三" |
